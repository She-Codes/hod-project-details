iHOD Application

Virginia House of Delegates Interactive legislation tracking application.

Development on this app was already in progress when I began working with the House of Delegates and the overall design had already been established. jQuery Mobile was used for the frontend as a solution for making the app accessible for the many delegates who access it using their House-issued iPads. The backend was built using Rails. In addition to delegates and their staff, the app is widely used throughout the organization to accomplish department-specific tasks during a legislative session. I contributed to building out certain department-specific features of the app including those used by the Enrolling and Tracking department and Committee Operations. I also contributed to the frontend functionality of some of the overall bill tracking features.

Agenda Form

One of the key features of iHOD is the ability to set agendas for committee meetings. A major component of the form is the ability to add 'blocks' of bills to a meeting agenda. An example of the Agenda Block section is pictured below.

Adding and Deleting Bills

Users are able to define 'blocks' that will then be addressed in committee meetings. The available committee bills are fetched from the API and cached. The user can select bills to add using the modal. When bills are deleted from the block they are again available for selection.

Sorting Bills in a Block

Custom written JavaScript allows the sorting of bills in a block. Bills are sorted alphabetically either by bill number or patron name. A custom sort is also available with the help of jQuery UI's drag and drop widget.

Adding and Deleting a Block

Not only bills, but blocks as well, can be added to an agenda and deleted.

Reordering Blocks

Because blocks can often get quite large, drag and drop was not feasible and would've made it more difficult for the user to reorder blocks due to the increase in dragging distance. Instead, the blocks can be reordered in the DOM one by one using arrow buttons.

Moving Bills Between Blocks

Again, because of the potentially large size of blocks, I decided against drag and drop for moving bills from one block to another. Once the bills to be moved are selected, all that's needed is to select the destination block.

Form Validation

The form is validated with Rails, but there is also JavaScript validation in place. Users can select block types, but Doc and Link blocks can not contain bills. As long as the block has bills, the block type can not be changed.

Enrolling Memo Form

The Enrolling and Tracking department needed specialized forms, as well. The form used to draft an enrolling memo is one example. A list of bills was fetched from the API and cached. Using mouse events bills are easily highlighted and selected. Bills of a specified type can be filtered using the checkbox on the left and all bills can be cleared using the button on the right.
