Purchase Order Application

Utilized by the Virginia House of Delegates Support Services to manage vendors and purchases.

This application was an update of an older system created with Microsoft Sharepoint. The new version is built with Ruby on Rails, Bootstrap, vanilla JavaScript where possible and a bit of jQuery sprinkled throughout.

Purchase Order Form

The primary users of the application were very involved in the design of the form. It is an updated version of a Sharepoint form that was created years ago and the user did not want to stray too far from the original design. The header of the page needed to remain very similar to the original as did other elements of the page, but fonts and other design elements were changed to give a more updated feel. The main highlights would be the custom JavaScript typeahead written to fuzzy search vendor names and the dynamic order area.

Custom Built Typeahead

Because the PO app contains sensitive information and actual vendor data is stored in the database, the Codepen to the right shows a demo of the same typeahead using almost the same code used in the app with data from an anime API. As with the widget in the app, you can use the up and down arrow keys to navigate the list of results and press the Enter key to select one. Alternatively you can use the mouse to scroll and select.

See the Pen vanilla js typeahead by Nikia Shaw (@she_codes) on CodePen.

Adding and Deleting Items

When 'Add Item' is clicked, a new row is added to the actual 'Purchase Order' section of the form. As numbers are added to the 'Qty' and 'Unit Cost' inputs, row totals, as well as a grand total, are calculated using JavaScript. An item can easily be deleted using the red trash can to the right. Item is actually a nested Rails model that is being handled within the Purchase Order form with the help of JavaScript.
